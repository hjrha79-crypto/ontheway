===== 이슈 1: 배민 묶음배달 중복 TTS 방지 =====

현상: 묶음 2건 콜이 들어오면 개별(6,420원) + 합산(8,720원) + 개별(2,300원) 3번 TTS됨.
원인: 배민 화면 스와이프 시 개별/합산이 각각 Accessibility 이벤트로 들어옴.

수정:
1. OnTheWayService에 배민 전용 debounce 로직 추가
2. 배민 콜 감지 시 즉시 TTS 하지 않고 2초 대기
3. 2초 윈도우 내 같은 플랫폼(배민) 콜을 리스트에 모음
4. 2초 후 판정:
   - 리스트 중 bundleCount > 1인 콜이 있으면 → 그 콜만 TTS + FilterLog 저장
   - bundleCount > 1인 콜이 없으면 → 마지막 콜만 TTS + FilterLog 저장
   - 나머지 콜은 FilterLog에만 저장 (TTS 없음, reason에 "묶음 중복 제거" 표기)
5. Handler + postDelayed 패턴 사용
6. 쿠팡은 debounce 적용 안 함 (기존 즉시 TTS 유지)

===== 이슈 2: 쿠팡이츠 Accessibility 진단 로그 =====

현상: 쿠팡이츠 앱 출근 상태, 콜 알림 울리는데 OnTheWay 완전 무반응. 접근성 서비스 ON 상태. 배민은 정상 감지됨.
의심: 쿠팡 앱 업데이트로 Accessibility 이벤트 패턴 변경 또는 getWindows()에서 패키지 못 잡음.

수정:
1. SettingsActivity에 "쿠팡 진단 모드" Switch 추가 (SharedPreferences 키: "coupang_debug_mode", 기본 OFF)
2. OnTheWayService.onAccessibilityEvent()에서:
   - 쿠팡 진단 모드 ON일 때
   - 모든 이벤트(패키지 무관)에서 packageName이 "com.coupang.mobile.eats.courier"이면:
     → FilterLog에 기록: platform="쿠팡진단", price="0", judgment="DEBUG",
        reason="eventType=[타입] pkg=[패키지명] text=[이벤트텍스트전체] class=[className]"
   - 추가로 getWindows()의 모든 윈도우 패키지명도 1회 로그:
     reason="WINDOWS: [윈도우1패키지, 윈도우2패키지, ...]"
3. 쿠팡 진단 모드 OFF면 기존 로직 그대로
