OnTheWay 프로젝트에서 v3.4 GPS 연동을 구현해줘:

1. 위치 권한 및 GPS 수신:
   - AndroidManifest.xml에 ACCESS_FINE_LOCATION, ACCESS_COARSE_LOCATION 권한 추가
   - OnTheWayService.kt에서 LocationManager를 사용하여 GPS 위치 수신
   - 30초마다 위치 업데이트 (배터리 절약)
   - 현재 위도/경도를 companion object에 저장 (currentLat, currentLng)
   - 위치 권한 없으면 GPS 기능 비활성화 (크래시 방지)

2. 현재위치 → 픽업지 거리 계산:
   - 배민/쿠팡 콜의 픽업지 주소에서 동/읍/면 이름을 추출
   - 주요 지역 좌표 테이블 생성 (LocationTable.kt):
     태전동(37.4095, 127.2555), 장지동(37.4020, 127.2480), 양벌동(37.3850, 127.2650),
     오포읍(37.3750, 127.2100), 중대동(37.4200, 127.2700), 탄벌동(37.4300, 127.2850),
     역동(37.4150, 127.2750), 광남동(37.4050, 127.2600), 송정동(37.4180, 127.2400)
     등 경기 광주시 주요 동 좌표 20개 이상
   - Haversine 공식으로 현재위치 → 픽업지 동 좌표 거리 계산
   - 픽업지까지 거리를 CallData에 pickupDistance로 추가

3. 총 이동 거리 기반 판정:
   - 총거리 = 현재위치→픽업지 + 픽업지→배달지(포인트 환산)
   - 총단가 = 금액 / 총거리
   - CallFilter에서 총단가 기준으로 판정
   - reason에 "픽업 N.Nkm + 배달 N.Nkm = 총 N.Nkm, 총단가 N,NNN원/km" 표시
   - 콜 상세 다이얼로그에도 픽업거리/총거리/총단가 표시

4. 자동 방향 판별:
   - 현재위치 → 배달지 방향을 자동 계산 (동서남북 8방향)
   - 귀가 방향 설정이 있으면, 배달지가 귀가 방향과 일치하는지 자동 판별
   - 수동 입력 없이도 "이 콜은 귀가 방향입니다" 자동 표시
   - reason에 "귀가 방향 일치" 또는 "역방향" 표시

5. 이동 속도 기반 예상 도착 시간:
   - GPS에서 현재 속도 수신 (location.speed)
   - 정지 상태면 평균 속도 30km/h로 가정
   - "픽업까지 약 N분" 예상 시간 계산
   - 콜 상세에 표시, TTS에 "픽업 N분 거리" 추가

6. 상태 탭에 현재 위치 표시:
   - 상태 탭 상단에 "현재 위치: 태전동 부근" 표시
   - GPS 수신 상태 아이콘 (수신중/미수신)
   - 설정: "GPS 위치 사용" 토글 (기본 OFF)

7. 위치 권한 요청:
   - MainActivity에서 앱 시작 시 GPS 토글이 ON인데 위치 권한이 없으면 권한 요청 다이얼로그 표시
   - 권한 거부 시 GPS 기능 자동 OFF

수정 후 git add -A && git commit -m "v3.4: GPS연동 - 픽업거리계산 + 총단가판정 + 자동방향판별 + 예상도착시간" && git push 해줘.
