OnTheWay 프로젝트에서 v3.0 선행 구현을 해줘. 모든 신규 기능은 설정에서 ON/OFF 토글로 제어하고, 기본값은 OFF (숨김 상태):

1. 수익 트래킹:
   - Accessibility로 "배차수락", "배차 수락", "주문 수락", "수락하기", "모두 수락" 버튼 클릭을 감지하면, 직전 판정된 콜의 금액을 "수락됨"으로 FilterLog에 기록
   - 오늘 수락 건수, 총 매출, 시급(매출/첫수락~마지막수락 시간), 목표 달성률을 계산하는 EarningsTracker.kt 싱글톤 생성
   - 상단 "OnTheWay 0원" 부분을 실시간 매출로 업데이트 (예: "OnTheWay 62,400원")
   - "0콜 · 0원/h · 목표 100,000원" 부분도 실시간 업데이트 (예: "17콜 · 15,600원/h · 목표 100,000원")
   - 설정: "수익 트래킹" 토글 (기본 OFF)

2. 네비 자동 연동:
   - 콜 수락 감지 시, 해당 콜의 픽업지 주소로 카카오내비 Intent 발사
   - Intent: "kakaomap://route?ep=주소" 또는 "nmap://route?..." 또는 일반 geo: Intent
   - 카카오내비 우선, 없으면 카카오맵, 없으면 기본 지도앱
   - 설정: "네비 자동실행" 토글 (기본 OFF), 네비앱 선택 (카카오내비/티맵/카카오맵)

3. 자동 수락:
   - "잡으세요" 판정된 콜만 자동으로 수락 버튼 클릭
   - 수락 전 TTS: "자동 수락합니다"
   - 수락 후 3초 대기 → 네비 자동실행 (2번이 ON이면)
   - 안전장치: 자동수락 후 60초 쿨다운 (연속 자동수락 방지)
   - 설정: "자동 수락 (잡으세요만)" 토글 (기본 OFF)

4. 방향 필터:
   - 설정에서 "귀가 방향" 주소 입력란 추가
   - 귀가 방향이 설정되면, 배달지가 귀가 방향과 같은 방향인 콜에 판정 보너스 (기준 -500원 완화)
   - 반대 방향이면 기준 +500원 강화
   - 방향 판단: 배달지 주소에 귀가 방향의 동/읍/면 이름이 포함되어 있는지로 간단 비교
   - reason에 "귀가 방향 보너스" 또는 "역방향 페널티" 표시
   - 설정: "귀가 방향 필터" 토글 (기본 OFF) + 주소 입력

5. 일별 리포트:
   - 매일 마지막 콜 감지 후 30분간 새 콜이 없으면 "오늘 요약" 알림 자동 생성
   - 내용: "오늘 N건 감지, M건 수락, 매출 N원, 시급 N원, 넘긴 콜 중 아쉬운 콜 N건"
   - "아쉬운 콜" = 넘긴 콜 중 단가가 가장 높았던 상위 3건
   - Android Notification으로 표시
   - 설정: "일별 리포트" 토글 (기본 OFF)

6. 설정 화면 구성:
   - 기존 설정에 "고급 기능" 섹션 추가
   - 토글 목록: 수익 트래킹, 네비 자동실행, 자동 수락, 귀가 방향 필터, 일별 리포트
   - 각 토글 아래에 한 줄 설명 표시
   - SharedPreferences로 저장

수정 후 git add -A && git commit -m "v3.0: 수익트래킹 + 네비연동 + 자동수락 + 방향필터 + 일별리포트 선행구현 (전체 토글OFF 기본)" && git push 해줘.
