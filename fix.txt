OnTheWay 프로젝트에서 다음을 모두 수정해줘. 이번은 v2 2.0 메이저 업데이트야:

1. 카카오T 감지 근본 수정:
   - accessibility_service_config.xml에서 android:packageNames 속성을 완전히 제거 (모든 앱 이벤트 수신)
   - accessibilityEventTypes를 typeAllMask로 변경
   - onAccessibilityEvent 최상단에서 "kakaomobility" 또는 "flexer" 포함 패키지면 Log.w("OTW_KAKAO", "★ 카카오T: pkg=$pkg, type=${event.eventType}, source=${event.source != null}, root=${rootInActiveWindow != null}") 진단 로그 추가
   - TARGET_PACKAGES 체크는 유지하되, 카카오T 이벤트가 들어오면 getWindows()로도 카카오T window를 찾아보는 로직 추가

2. 배민 getWindows() 백그라운드 감지:
   - onAccessibilityEvent에서 배민 이벤트가 들어왔는데 rootInActiveWindow의 패키지가 배민이 아닌 경우, getWindows()로 모든 window를 순회하여 배민 콜 화면을 찾아서 처리

3. 배민 비콜 화면 필터 강화 - 다음 키워드가 포함된 화면은 파싱 스킵:
   - "가상 배달을 체험해 보세요", "배달 체험하기"
   - "진행 배달미션", "배달 미션", "완료 시 최대", "미션 전체보기"
   - "메뉴금액", "주문정보", "가게정보", "찾아오는 길"
   - "신규배차를 켜고", "배달을 시작해", "배차 대기"

4. 묶음배달 판정 기준 건수 비례 상향:
   - 1건: 최소 3,000원 / 고액 7,000원
   - 2건: 최소 5,500원 / 고액 10,000원
   - 3건: 최소 8,000원 / 고액 13,000원
   - 공식: 최소 = 3000 + (건수-1)*2500, 고액 = 7000 + (건수-1)*3000
   - reason에 "묶음 N건, 건당 N,NNN원" 포함

5. 다중 픽업 묶음 추가 기준:
   - 픽업지(가게명)가 서로 다른 묶음이면 기준 추가 상향
   - 다중 픽업 2건: 최소 7,000원
   - 다중 픽업 3건: 최소 10,000원
   - reason에 "다중 픽업" 표시

6. 묶음배달 부분 파싱 중복 방지:
   - "총 합계" 파싱 후 10초 이내에 같은 콜의 부분 조합이 감지되면 스킵
   - TtsDeduplicator에 묶음 총액 기록하고, 해당 시간 내 부분합이 들어오면 무시

7. 배민 포인트 기반 거리 환산 판정:
   - BaeminParser에서 포인트(P) 값을 파싱하여 CallData에 포함
   - 포인트 1P = 0.15km 환산
   - 환산 단가 = 금액 / (포인트 * 0.15)
   - 단가 2,000원/km 미만이면 REJECT
   - 포인트 구간별 최소금액: 15P이하 3,000원, 15~25P 4,000원, 25P이상 5,000원
   - reason에 "포인트 N.NP (환산 N.Nkm), 단가 N,NNN원/km" 포함

8. 콜 상세 다이얼로그에 포인트/환산거리/단가 표시:
   - 포인트가 있으면: "포인트: 30.8P (환산 4.6km)"
   - 단가가 있으면: "단가: 1,013원/km"

9. "잡으세요" 판정 기준 확장:
   - 기존: 단가 ≥ 2,500원/km + 거리 ≤ 3km
   - 추가: 금액 ≥ 10,000원이면 무조건 "잡으세요"
   - 추가: 금액 ≥ 7,000원 + (거리 ≤ 3km 또는 포인트 ≤ 15P)이면 "잡으세요"

10. 접근성 서비스 체크: MainActivity에서 앱 시작 시 접근성 서비스와 알림 리스너가 꺼져있으면 상단에 경고 배너 표시
