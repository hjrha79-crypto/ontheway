OnTheWay 프로젝트에서 카카오T 감지 진단 기능을 추가해줘:

1. OnTheWayService.kt의 onAccessibilityEvent 최상단에서 모든 패키지의 이벤트를 카운트하는 진단 모드 추가:
   - packageEventCount라는 HashMap<String, Int>을 만들어서, 수신되는 모든 패키지별 이벤트 수를 카운트
   - 카카오 관련 패키지(kakaomobility, flexer, kakao)는 별도로 Log.w 출력

2. 설정 화면에 "진단 모드" 버튼 추가:
   - 버튼 누르면 AlertDialog로 현재까지 수신된 패키지별 이벤트 수 표시
   - "com.woowahan.bros: 342건, com.coupang.mobile.eats.courier: 128건, com.kakaomobility.flexer: 0건" 형태
   - 카카오T가 0건이면 Accessibility 이벤트 자체가 안 오는 것 확정

3. 카카오T가 0건일 경우를 대비한 대안:
   - DeliveryNotificationService.kt에서 카카오T 픽커(com.kakaomobility.flexer) 알림도 수신하도록 추가
   - 카카오T 알림 title/text 구조를 로깅하여 금액 파싱 가능 여부 확인
   - 알림에 금액이 있으면 NotificationListener로 처리

수정 후 git add -A && git commit -m "v2.2: 카카오T 진단모드 + NotificationListener 대안 경로 추가" && git push 해줘.
