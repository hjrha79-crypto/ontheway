OnTheWay v2 저녁 수정 — 9개 항목 일괄 구현

현재 상태: 3플랫폼(배민/쿠팡/카카오T) 감지 + "넘겨라" TTS 작동 확인됨

═══ 수정 1: FilterLog 저장 연결 확인 ═══
FilterLog.kt에 save 함수가 있는지, OnTheWayService에서 실제로 호출되는지 확인.
안 되어 있으면 연결. 모든 콜(REJECT+ACCEPT)에서 저장 필수.
저장 필드: timestamp, platform, rawText, price, distanceKm, unitPrice, isMulti, result, parseSuccess, storeName, destination

═══ 수정 2: 카카오T "P" 금액 파싱 검증 ═══
카카오T 픽커 콜은 "3,200P" "4,350P" 형태. 기존 amountPattern이 "원"만 매칭하면 "P"도 추가.
카카오T는 픽업거리 + 배송거리 합산으로 단가 계산.

═══ 수정 3: 대기 화면 무시 필터 ═══
배민: "가상 배달을 체험해 보세요", "신규배차를 켜고 배달을 시작하세요" 포함 시 무시
쿠팡: 텍스트가 비어있거나 앱 목록(NAVER, YouTube 등 나열) 포함 시 무시
→ 파싱 시도 자체를 스킵하고 로그도 안 남김

═══ 수정 4: TTS 3단계 판정 ═══
REJECT: "넘기세요" → 조건: 최소금액 미달 또는 단가 미달
ACCEPT 상위: "잡으세요" → 조건: 단가 2500원/km 이상 AND 거리 있는 플랫폼에서 배달거리 3km 이하
ACCEPT 일반: "괜찮습니다" → 나머지 ACCEPT

TTS 형식:
"배민, 넘기세요, 이천오백원"
"쿠팡, 잡으세요, 단가 삼천이백"
"쿠팡, 괜찮습니다, 단가 이천오백 픽업 멉니다"
"카카오, 괜찮습니다, 단가 이천백"

배민은 거리 없으므로 "잡으세요" 불가, 최대 "괜찮습니다"
배민 포인트 25P 이상이면: "괜찮습니다, 먼 거리입니다"

설정에서 "괜찮습니다" 음성 ON/OFF 가능

═══ 수정 5: "잡아" 음성 수락 ═══
VoiceManager 또는 SpeechRecognizer로 "잡아"/"수락"/"이거" 감지
→ 현재 포그라운드 앱에서 수락 버튼 자동 클릭
→ TTS "수락합니다"

수락 버튼 텍스트 매칭: "배차수락", "배차 수락", "주문 수락", "주문수락", "수락하기", "모두 수락" (contains 매칭)
2초 타임아웃: 마지막 콜 감지 후 30초 이내만 수락 가능

═══ 수정 6: 메인 화면 상태 표시 ═══
MainActivity 상단 (수익 영역 아래)에 상태 카드:
"필터 작동 중 · 마지막 감지 N초 전"
"오늘 N건 (넘기세요 X · 괜찮습니다 Y · 잡으세요 Z)"
FilterLog에서 읽어서 표시. 5초마다 갱신.

═══ 수정 7: 설정 화면 추가 항목 ═══
- 필터 기록: 최근 20건 리스트 (시간/플랫폼/금액/단가/판정)
- 오늘 요약: REJECT 평균 금액 / ACCEPT 평균 금액
- "괜찮습니다" 음성 ON/OFF 토글
- 최대 픽업 거리 설정 (기본 5km) — 쿠팡/카카오T에서 배달거리 초과 시 REJECT

═══ 수정 8: 배민 포인트 거리 경고 ═══
배민커넥트 포인트 파싱 추가: "[\d.]+P" 매칭
25P 이상이면 TTS에 "먼 거리입니다" 추가

═══ 수정 9: FilterLog 필드 확장 ═══
기존 필드 + storeName(가게명) + destination(전달지) 추가 저장
향후 👍/👎 기능 대비
